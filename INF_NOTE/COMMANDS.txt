Volatility 3 does not have a plugin called windows.registry.hashdump.Hashdump. That was a Volatility 2 plugin.
In Volatility 3, the equivalent functionality is split into different plugins under windows.registry.*. For extracting user hashes (SAM + SYSTEM), you should use:
windows.registry.hivelist.HiveList   (A list of Windows Registry hive files loaded in memory RAM {SYSTEM, SOFTWARE, SAM, SECURITY, NTUSER.DAT})
windows.registry.getcellroutine.GetCellRoutine      (An internal Windows Registry function that retrieves a registry cell (data structure) from memory for processing.)
windows.registry.printkey.PrintKey   (breakdown of hivelist)(A Volatility plugin used to display registry keys and their values from a specific registry hive.)

⭐ What is a mutex (exam definition)
A mutex (mutant) is:
👉 A Windows synchronization object
👉 Used to ensure only one process accesses a resource
BUT…

🔥 Malware commonly creates unique mutex names to:
prevent reinfection
coordinate malware threads
identify active infection
So mutex names are strong malware indicators.

Windows artifacts are digital traces left on a Windows system that provide evidence of user and system activity, used in digital forensic investigations.
Simple Classification Table
| Artifact Type | What it shows         |
| ------------- | --------------------- |
| File system   | File activity         |
| Registry      | Configuration & usage |
| User activity | User interaction      |
| Execution     | Program runs          |
| Logs          | System events         |
| Web/network   | Internet activity     |
| Memory        | Live system state     |

In cybersecurity, an artifact is any piece of digital data that provides evidence of user activity, system interactions, or malicious behavior, 
crucial for forensic investigations and incident analysis.
Definition and Scope
An artifact in cybersecurity refers to data or traces left behind on digital systems that can reveal actions performed by users, applications, or attackers. 
These are not limited to files; they include system logs, memory dumps, registry entries, browser histories, file timestamps, network traffic captures, metadata, 
and remnants of deleted items. Artifacts serve as digital footprints that help investigators reconstruct events and understand the sequence of actions on a device 
or network.

⭐What is a hidden process?
Exists in memory but does NOT appear in the standard active process list (pslist), OR
Appears in pslist but has abnormal attributes (wrong parent, wrong session, no handles), OR
Is only found in psscan (raw memory scan for EPROCESS structures) but not in pslist.
Present in psscan (memory scan)
Missing from pslist (standard process enumeration)

Which is the Parent process and (PPID) of System process?
Parent Process: None (PPID = 0)
Reasoning: The System process (PID 4) is a core Windows kernel process. Its parent PID is 0, which indicates it is directly started by the Windows kernel 
at boot and has no “user-mode” parent process.
| Process Name | PID | PPID | Notes                                     |
| ------------ | --- | ---- | ----------------------------------------- |
| System       | 4   | 0    | Kernel process, parent is kernel (PPID 0) |
