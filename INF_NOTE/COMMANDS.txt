Volatility 3 does not have a plugin called windows.registry.hashdump.Hashdump. That was a Volatility 2 plugin.
In Volatility 3, the equivalent functionality is split into different plugins under windows.registry.*. For extracting user hashes (SAM + SYSTEM), you should use:
windows.registry.hivelist.HiveList   (A list of Windows Registry hive files loaded in memory RAM {SYSTEM, SOFTWARE, SAM, SECURITY, NTUSER.DAT})
windows.registry.getcellroutine.GetCellRoutine      (An internal Windows Registry function that retrieves a registry cell (data structure) from memory for processing.)
windows.registry.printkey.PrintKey   (breakdown of hivelist)(A Volatility plugin used to display registry keys and their values from a specific registry hive.)
