## ✅ E) Which Process is connected to a Private Network?
### 👉 Command
```bash
python3 vol.py -f Win11Dump.mem windows.netscan
```
### 👉 What to look for
Private IP ranges:
| Range       | Meaning |
| ----------- | ------- |
| 10.x.x.x    | Private |
| 172.16–31   | Private |
| 192.168.x.x | Private |
Example:
```
chrome.exe 192.168.1.10 → 192.168.1.1
```
### ✅ Exam Answer
> The process **chrome.exe** is connected to a private network.
### ✅ Motivation
> Netscan output shows connection using private IP range 192.168.x.x which is RFC1918 private addressing.

## ✅ H) Hidden processes + hiding method ⭐ (HIGH MARK QUESTION)
### 👉 Commands
```bash
python3 vol.py -f WinDump.mem windows.pslist
python3 vol.py -f WinDump.mem windows.psscan
```
### 👉 Method
Compare:
| pslist      | psscan    |
| ----------- | --------- |
| Linked list | Pool scan |

If process only in **psscan → hidden**
### ✅ Exam Answer
> Process evil.exe was detected in psscan but not pslist.
### ✅ Motivation (FULL MARKS)
> This indicates DKOM unlinking where malware removes its EPROCESS from the active process list to evade detection.

## ✅ O) 20-word mutex explanations (COPY THESE)
1️⃣
> Ensures only one malware instance runs simultaneously to prevent conflicts and maintain persistence on infected systems.
2️⃣
> Used by applications to coordinate shared resource access and avoid simultaneous execution conflicts.
3️⃣
> Allows malware to detect existing infections and terminate duplicate executions automatically.
4️⃣
> Synchronization object controlling process access to system resources preventing race conditions.
