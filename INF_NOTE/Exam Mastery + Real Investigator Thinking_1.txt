# ⭐ 1️⃣ EXAM ANSWER TEMPLATES (COPY-PASTE)
Use this structure for every question.

## ✅ TEMPLATE — IDENTIFICATION QUESTION:
Plugin used:
windows.info.Info

Evidence:
The plugin output shows OS version/build information.

Interpretation:
This identifies the operating system used in the memory image.

Conclusion:
The system is running <INSERT OS>.

## ✅ TEMPLATE — PROCESS QUESTION:
Plugin used:
windows.pslist.PsList

Evidence:
The plugin lists active processes including <PROCESS>.

Interpretation:
This confirms that <PROCESS> was active at acquisition time.

Conclusion:
The system had <PROCESS> running.

## ✅ TEMPLATE — HIDDEN PROCESS QUESTION
Plugin used:
windows.pslist.PsList
windows.psscan.PsScan

Evidence:
PID <X> appears in PsScan but not in PsList.

Interpretation:
This indicates the process was unlinked from active lists.

Security implication:
This behaviour is consistent with rootkit hiding techniques.

Conclusion:
PID <X> is a hidden process.

## ✅ TEMPLATE — NETWORK QUESTION
Plugin used:
windows.netscan.NetScan

Evidence:
PID <X> connected to IP <IP>.

Interpretation:
This indicates network communication.

Security implication:
Connection to external/private address suggests possible attacker activity.

Conclusion:
Process <X> maintained network connectivity.

## ✅ TEMPLATE — USER QUESTION
Plugin used:
windows.sessions.Sessions

Evidence:
User <NAME> logged into session.

Interpretation:
This indicates interactive logon.

Conclusion:
User <NAME> was active.

## ✅ TEMPLATE — MALWARE QUESTION
Plugin used:
windows.malfind.Malfind

Evidence:
Executable memory region with PAGE_EXECUTE_READWRITE.

Interpretation:
This suggests injected code.

Security implication:
Strong evidence of malware.

Conclusion:
Process <X> contains injected code.

# ⭐ 2️⃣ WIN DUMP QUESTIONS — STEP-BY-STEP SOLUTION METHOD

### Command
python3 vol.py -f Win11Dump.mem windows.info.Info

### What to answer
👉 Look at header:
Volatility 3 Framework X.X

### Motivation
> The framework version is displayed in plugin output header.

# ✅ E) Process connected to private network
### Command
windows.netscan.NetScan

### What to look for
Private ranges:
 10.x.x.x
 172.16-31.x.x
 192.168.x.x

### Motivation
> PID X connected to 192.168.x.x which falls within RFC1918 private addressing.
