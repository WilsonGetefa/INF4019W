

Step-by-step guide to connect GitHub with Kali Linux and VS Code.
✅ Clone repositories
✅ Push code to GitHub
✅ Use VS Code Git integration
✅ Authenticate securely (SSH recommended)

# ✅ STEP 1 — Install Git in Kali
Most Kali systems already have Git.
Check:
```bash
git --version
```

If missing:
```bash
sudo apt update
sudo apt install git -y
```

# ✅ STEP 2 — Configure Git identity (VERY IMPORTANT)
This tells Git who you are.
```bash
git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"
```

Check:
```bash
git config --list
```
👉 Use the same email as GitHub


# ✅ STEP 3 — Create SSH key for GitHub (BEST METHOD)
### Generate key
```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Press Enter → Enter → Enter
Key saved in:
```
~/.ssh/id_ed25519
~/.ssh/id_ed25519.pub
```

# ✅ STEP 4 — Start SSH agent
```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

# ✅ STEP 5 — Copy public key
```bash
cat ~/.ssh/id_ed25519.pub
```

# ✅ STEP 6 — Add key to GitHub
1. Open → GitHub
2. Settings
3. SSH and GPG keys
4. New SSH key
5. Paste key
6. Save

# ✅ STEP 7 — Test connection
```bash
ssh -T git@github.com
```
Expected:
```
Hi username! You've successfully authenticated
```

# ✅ STEP 8 — Clone repository in Kali
Example:
```bash
git clone git@github.com:username/repo.git
```
Now repo is in Kali.

# ✅ STEP 9 — Open repo in VS Code
```bash
cd repo
code .
```
👉 VS Code auto-detects Git.
You will see:
 Source control tab
 Commit button
 Push/pull buttons


# ✅ STEP 10 — Commit & push workflow
### Add files
```bash
git add .
```

### Commit
```bash
git commit -m "Initial commit"
```
### Push
```bash
git push
```

# ⭐ OPTIONAL — Login GitHub inside VS Code UI
Inside VS Code:
1. Bottom left → Accounts
2. Sign in to GitHub
3. Browser login
4. Done

👉 This enables:
 Pull requests
 GitHub extensions
 Repo browsing

# 🔥 Common Kali mistakes
### ❌ Permission denied (publickey)
Fix:
```bash
ssh-add ~/.ssh/id_ed25519
```

### ❌ HTTPS password no longer works
GitHub removed password auth.
👉 Use SSH or Personal Access Token
---
### ❌ Repo cloned but cannot push
Check remote:
```bash
git remote -v
```

If HTTPS → change to SSH:
```bash
git remote set-url origin git@github.com:username/repo.git
```

Yes ✅ — **that is exactly how GitHub is meant to work.**

After creating your repository on GitHub, you can:

1️⃣ Download (clone) the current project
2️⃣ Add new files
3️⃣ Push the changes back to GitHub

Let me show you the proper workflow.

---

# 🔹 Step 1 — Clone the existing GitHub project

If using SSH (recommended):

```bash
git clone git@github.com:username/project-name.git
```

If using HTTPS:

```bash
git clone https://github.com/username/project-name.git
```

This downloads the project to your Kali machine.

---

# 🔹 Step 2 — Enter the project folder

```bash
cd project-name
```

Open in VS Code (since you're using Kali + VS Code):

```bash
code .
```

---

# 🔹 Step 3 — Add your new file

Example:

```bash
touch newscript.py
```

Or create/edit files in VS Code.

---

# 🔹 Step 4 — Add the file to Git

```bash
git add newscript.py
```

Or add everything:

```bash
git add .
```

---

# 🔹 Step 5 — Commit your changes

```bash
git commit -m "Added new forensic script"
```

---

# 🔹 Step 6 — Push to GitHub

```bash
git push
```

Now your new file appears in your repository on GitHub.

---

# 🔥 Important Things to Remember

## ⚠️ Always clone first

Never just create a new folder and try to push unless you initialized Git properly.

---

## ⚠️ If someone updated the repo before you

Always pull first:

```bash
git pull
```

# 🚫 Common Mistake

If you get:

```
fatal: not a git repository
```

It means you are not inside the cloned folder.

Run:

```bash
ls -a
```

You should see a `.git` folder.
